# Grill-intent SOP (§7.6)

Run BEFORE planning, on the user's verbatim words only — no paraphrase, no pre-stuffing, no thoroughness directives. Capturing the user verbatim is a fidelity rule on **their** words — not a gag on yours: you are a reasoning partner here, not a scribe.

1. **One question at a time.** Ask the single most important unresolved question, with options and a recommendation. Wait. Then the next. Never batch Q1/Q2/Q3.
2. **Drive every brief section to resolved-or-waived:** problem, feature, user_stated criteria/constraints, acceptance criteria (phrased as end-to-end observable behavior), out_of_scope. A waived section is recorded as waived — never silently empty.
3. **Attribution discipline:** the user's statements go to `user_stated` verbatim-faithfully. Your inferences — and any materially-better approach you see (outcome-changing only, never taste) — go to `conductor_proposals`, each marked confirmed/unconfirmed by an explicit user response. An unanswered recommendation is NOT an accepted one. Same rule for **limitations**: any limitation the chosen approach would carry — especially one that weakens or defeats the feature's purpose (it "works" but can't do what it's *for*) — is surfaced as an explicit **keep-or-solve question** and recorded confirmed/waived in the brief. A limitation you foresaw but did not raise is NOT an accepted one.
4. **Research trigger:** "can we do X?" that the user cannot answer → dispatch the researcher mid-grill (bounded question, capped budget); never guess, never park it silently.
5. Output: brief sections written via `knowledge_create`/update on the draft brief; open proposals listed back to the user for explicit accept/reject.
